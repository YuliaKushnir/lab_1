## Лабораторна робота №1

### Тема: дослідження роботи з елементами керування

Для виконання лабораторної роботи було створено Android-проєкт у середовищі Android Studio за шаблоном Empty Views Activity. 

За допомогою перетягування компонентів на робочу область у файлі розмітки activity_main.xml було створено інтерфейс, який містить TextView для відображення заголовка, RadioGroup з трьома RadioButton для вибору діаметра (оскільки діаметр має бути обраний лише один, RadioButton більше підходить),  три CheckBox для вибору додаткових опцій, Button для підтвердження вибору, TextView для відображення результату. 
 ![img_1.png](img_1.png)

Для обробки натискання кнопки використано метод onClickOk. При натисканні він:
-  перевіряє чи вибраний діаметр (radioGroup.getCheckedRadioButtonId()), 
- перевіряє чи вибрана хоча б одна додаткова опція (checkBox.isChecked()),
- якщо дані введені не повністю – показує Toast з попередженням «Виберіть хоча б одну додаткову опцію» (Toast.makeText())
- якщо всі дані введені – результат виводить у TextView(textView.setText()).

Для коректного запуску змінено налаштування для емулятора: збільшено RAM до 4 Гб, VM heap size – до 1 Гб. 

Коректність роботи програми відображено на скріншотах нижче. 

1) Якщо не обрати жоден з чек-боксів та радіобатон – виникає спливаюче вікно з відповідним повідомленням: 
![img_2.png](img_2.png) 

2) Якщо обрано хоча б один з чек-боксів-додаткових опцій та встановлено діаметр, результат успішно відображається у TextView:
 ![img_3.png](img_3.png)


### Висновок:
Під час виконання лабораторної роботи було вивчено принцип створення графічного інтерфейсу в Android Studio. Було реалізовано взаємодію між елементами інтерфейсу, такими як RadioButton, CheckBox, Button та TextView.
Також було освоєно використання обробників подій, перевірку введених даних та відображення повідомлень за допомогою Toast.
У результаті створено програму, яка коректно обробляє вибір користувача та відображає результат роботи інтерфейсу.
